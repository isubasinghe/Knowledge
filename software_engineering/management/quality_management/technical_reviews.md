# Technical Reviews

## Basic idea

Human inspection of an artefact (doc, design, code) to find defects. Works on *any* artefact — including non-executable ones that testing cannot touch — and catches faults testing misses.

## Key facts

- Earlier than testing in the lifecycle ⇒ cheaper fixes.
- Types: informal, formal, walk-throughs, code inspections, audits.
- Costs: time, planning, discipline.

## Technical Reviews

### Advantages of technical reviews

- Can be performed on any software artefact., whereas testing and measurement can only be performed on executable artefacts.
- Earlier detection of problems
- Review techniques find several types of faults, testing fails to find.

### Disadvantages

- Time and resource limiting
- Should be carefully planned and executed

### Types of technical reviews

- Informal
- Formal
- Walk throughs
- Code inspections
- Audits
