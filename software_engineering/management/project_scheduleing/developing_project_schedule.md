# Developing Project Schedule

## Basic idea

Five-step procedure: break work into tasks (WBS) → identify dependencies (task network) → estimate effort and time per task → allocate resources → produce a schedule (Gantt / PERT).

## Key facts

- WBS = hierarchical decomposition of project scope into manageable tasks.
- Task network is a DAG; critical path determines the project length.
- Resource allocation may extend the schedule beyond critical path due to contention.

## Developing Project Schedule
1. Breakdown the task into small chunks you can deal with **Work Breakdown Structure**
2. Identify the **interdependnecies** between tasks and develop a **task network**.
3. Estimate the **effort** and the **time allocation** for each task.
4. **Allocate resources** for tasks and validate effort
5. Develop the **project schedule**
